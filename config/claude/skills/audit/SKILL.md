体系的な品質チェックを実行し、優先度付けされた問題と実行可能な推奨事項を含む包括的な監査レポートを生成します。問題を修正するのではなく、他のコマンドで対処できるよう文書化します。

**最初に**: デザイン原則とアンチパターンについてfrontend-designスキルを使用してください。

## 診断スキャン

複数の次元にわたって包括的なチェックを実行する：

1. **アクセシビリティ（A11y）** - 以下をチェック：
   - **コントラストの問題**: テキストのコントラスト比 < 4.5:1（AAAの場合は7:1）
   - **ARIA不足**: 適切なロール、ラベル、状態のないインタラクティブ要素
   - **キーボードナビゲーション**: フォーカスインジケーターの欠如、非論理的なタブ順序、キーボードトラップ
   - **セマンティックHTML**: 不適切な見出し階層、ランドマークの欠如、ボタンの代わりにdivを使用
   - **alt テキスト**: 画像の説明が欠落または不適切
   - **フォームの問題**: ラベルなしの入力、不適切なエラーメッセージ、必須インジケーターの欠如

2. **パフォーマンス** - 以下をチェック：
   - **レイアウトスラッシング**: ループ内でのレイアウトプロパティの読み書き
   - **高コストなアニメーション**: transform/opacityではなくレイアウトプロパティ（width、height、top、left）のアニメーション
   - **最適化の欠如**: 遅延読み込みなしの画像、未最適化のアセット、will-changeの欠如
   - **バンドルサイズ**: 不要なインポート、未使用の依存関係
   - **レンダリングパフォーマンス**: 不要な再レンダリング、メモ化の欠如

3. **テーマ** - 以下をチェック：
   - **ハードコードされた色**: デザイントークンを使用していない色
   - **壊れたダークモード**: ダークモードバリアントの欠如、ダークテーマでの低コントラスト
   - **一貫性のないトークン**: 間違ったトークンの使用、トークンタイプの混在
   - **テーマ切り替えの問題**: テーマ変更時に更新されない値

4. **レスポンシブデザイン** - 以下をチェック：
   - **固定幅**: モバイルで崩れるハードコードされた幅
   - **タッチターゲット**: 44x44px未満のインタラクティブ要素
   - **水平スクロール**: 狭いビューポートでのコンテンツオーバーフロー
   - **テキストスケーリング**: テキストサイズ増加時に崩れるレイアウト
   - **ブレークポイントの欠如**: モバイル/タブレットバリアントなし

5. **アンチパターン（重要）** - frontend-designスキルの**DON'T**ガイドラインすべてをチェック。AIスロップの特徴（AIカラーパレット、グラデーションテキスト、グラスモーフィズム、ヒーローメトリクス、カードグリッド、汎用フォント）と一般的なデザインアンチパターン（色の上のグレー、ネストされたカード、バウンスイージング、冗長なコピー）を確認。

**重要**: これは監査であり、修正ではありません。影響の明確な説明を含め、問題を徹底的に文書化してください。監査後、問題の修正には他のコマンド（normalize、optimize、hardenなど）を使用してください。

## 包括的レポートの生成

以下の構造で詳細な監査レポートを作成する：

### アンチパターン判定
**ここから始める。** 合格/不合格：これはAI生成に見えるか？スキルのアンチパターンセクションから具体的な特徴をリストする。残酷なまでに正直に。

### エグゼクティブサマリー
- 見つかった問題の総数（重要度別のカウント）
- 最も重大な問題（上位3-5件）
- 全体的な品質スコア（該当する場合）
- 推奨される次のステップ

### 重要度別の詳細な発見事項

各問題について文書化：
- **場所**: 問題が発生する場所（コンポーネント、ファイル、行）
- **重要度**: 致命的 / 高 / 中 / 低
- **カテゴリー**: アクセシビリティ / パフォーマンス / テーマ / レスポンシブ
- **説明**: 問題の内容
- **影響**: ユーザーへの影響
- **WCAG/標準**: 違反している標準（該当する場合）
- **推奨事項**: 修正方法
- **推奨コマンド**: 問題の修正に適したスキルコマンド（利用可能なスキル一覧から選択する）

#### 致命的な問題
[コア機能をブロックするかWCAG Aに違反する問題]

#### 高重要度の問題
[重大なユーザビリティ/アクセシビリティへの影響、WCAG AA違反]

#### 中重要度の問題
[品質の問題、WCAG AAA違反、パフォーマンスの懸念]

#### 低重要度の問題
[軽微な不整合、最適化の機会]

### パターンと体系的な問題

繰り返し発生する問題を特定：
- 「ハードコードされた色が15以上のコンポーネントに存在、デザイントークンを使用すべき」
- 「タッチターゲットがモバイル体験全体で一貫して小さすぎる（44px未満）」
- 「すべてのカスタムインタラクティブコンポーネントでフォーカスインジケーターが欠如」

### ポジティブな発見事項

うまく機能していることを記録：
- 維持すべき良い慣行
- 他の場所で再現すべき模範的な実装

### 優先度別の推奨事項

実行可能な計画を作成：
1. **即時**: 最初に修正すべき致命的なブロッカー
2. **短期**: 高重要度の問題（今スプリント）
3. **中期**: 品質改善（次スプリント）
4. **長期**: あると良いもの、最適化

### 修正のための推奨コマンド

問題を利用可能なスキルコマンドにマッピングする。

例：
- 「デザインシステムとの整合のために`/normalize`を使用（N件のテーマ問題に対応）」
- 「パフォーマンス改善のために`/optimize`を使用（N件のパフォーマンス問題に対応）」
- 「耐障害性改善のために`/harden`を使用（N件のエッジケースに対応）」

**重要**: 徹底的だが実行可能であること。低優先度の問題が多すぎるとノイズになります。本当に重要なことに焦点を当ててください。

**絶対にしてはいけないこと**：
- 影響を説明せずに問題を報告する（なぜ重要なのか？）
- 重要度レベルを一貫性なく混在させる
- ポジティブな発見事項を省略する（うまく機能しているものを称える）
- 汎用的な推奨事項を提供する（具体的で実行可能に）
- 優先度付けを忘れる（すべてが致命的にはなれない）
- 検証なしに偽陽性を報告する

覚えておいてください：あなたは卓越した注意力を持つ品質監査人です。体系的に文書化し、容赦なく優先度付けし、改善への明確な道筋を示してください。良い監査は修正を容易にします。
