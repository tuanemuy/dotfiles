このスキルは、汎用的な「AIスロップ」美学を避けた、独自性のある、プロダクション品質のフロントエンドインターフェースの作成をガイドします。美的ディテールとクリエイティブな選択に卓越した注意を払った、実際に動作するコードを実装します。

## デザインの方向性

**大胆な**美的方向性にコミットする：
- **目的**: このインターフェースはどんな問題を解決するか？誰が使うか？
- **トーン**: 極端を選ぶ：過激なミニマル、マキシマリストな混沌、レトロフューチャリスティック、オーガニック/ナチュラル、ラグジュアリー/洗練、プレイフル/トイライク、エディトリアル/マガジン、ブルータリスト/生、アールデコ/ジオメトリック、ソフト/パステル、インダストリアル/ユーティリタリアンなど。選べるフレーバーはたくさんあります。インスピレーションとして使いつつ、美的方向性に忠実なものをデザインしてください。
- **制約**: 技術要件（フレームワーク、パフォーマンス、アクセシビリティ）。
- **差別化**: これを**忘れられないもの**にするのは何か？誰かが覚えている一つのことは？

**重要**: 明確なコンセプトの方向性を選び、精密に実行してください。大胆なマキシマリズムも洗練されたミニマリズムも機能します—鍵は意図性であり、強度ではありません。

次に、以下の特徴を持つ動作するコードを実装する：
- プロダクション品質で機能的
- ビジュアル的に印象的で記憶に残る
- 明確な美的ポイント・オブ・ビューとの一貫性
- すべてのディテールで綿密に洗練

## フロントエンド美学ガイドライン

### タイポグラフィ
→ *スケール、ペアリング、ロード戦略については[タイポグラフィリファレンス](reference/typography.md)を参照。*

美しく、ユニークで、興味深いフォントを選んでください。独自性のあるディスプレイフォントと洗練されたボディフォントをペアリングしてください。

**DO**: 流動的なサイジング（clamp）によるモジュラータイプスケールを使用する
**DO**: フォントウェイトとサイズを変えて明確なビジュアル階層を作る
**DON'T**: 使い古されたフォントを使用する—Inter、Roboto、Arial、Open Sans、システムデフォルト
**DON'T**: 「技術的/開発者」の雰囲気の安易な表現としてモノスペースタイポグラフィを使用する
**DON'T**: すべての見出しの上に丸角の大きなアイコンを置く—価値を追加することは稀で、テンプレート的に見える

### 色とテーマ
→ *OKLCH、パレット、ダークモードについては[カラーリファレンス](reference/color-and-contrast.md)を参照。*

まとまりのあるパレットにコミットする。ドミナントカラーとシャープなアクセントの方が、控えめで均等に分散されたパレットより効果的。

**DO**: 知覚的に均一で保守しやすいパレットにモダンCSSカラー関数（oklch、color-mix、light-dark）を使用する
**DO**: ニュートラルをブランドの色相に向けてティントする—微妙なヒントでも無意識の一貫性を生む
**DON'T**: 色付き背景にグレーテキストを使用する—洗い流されて見える；代わりに背景色の色合いを使用する
**DON'T**: ピュアブラック（#000）やピュアホワイト（#fff）を使用する—常にティントする；ピュアブラック/ホワイトは自然には存在しない
**DON'T**: AIカラーパレットを使用する：ダーク上のシアン、パープルトゥーブルーのグラデーション、ダーク背景のネオンアクセント
**DON'T**: 「インパクト」のためにグラデーションテキストを使用する—特にメトリクスや見出しに；装飾的で意味がない
**DON'T**: グローイングアクセント付きダークモードにデフォルトする—実際のデザイン判断を必要とせずに「クール」に見える

### レイアウトとスペース
→ *グリッド、リズム、コンテナクエリについては[空間リファレンス](reference/spatial-design.md)を参照。*

変化のあるスペーシングでビジュアルリズムを作る—どこでも同じパディングではなく。非対称と予想外の構図を受け入れる。強調のために意図的にグリッドを破る。

**DO**: 変化のあるスペーシングでビジュアルリズムを作る—タイトなグルーピング、ゆとりのある分離
**DO**: 大きな画面で呼吸するclamp()による流動的スペーシングを使用する
**DO**: 非対称と予想外の構図を使用する；強調のために意図的にグリッドを破る
**DON'T**: すべてをカードで包む—すべてにコンテナは不要
**DON'T**: カードの中にカードをネストする—視覚的ノイズ、階層をフラットにする
**DON'T**: 同一のカードグリッドを使用する—アイコン＋見出し＋テキストの同じサイズのカード、際限なく繰り返す
**DON'T**: ヒーローメトリクスレイアウトテンプレートを使用する—大きな数字、小さなラベル、サポート統計、グラデーションアクセント
**DON'T**: すべてをセンタリングする—左揃えテキストと非対称レイアウトの方がデザインされた感じがする
**DON'T**: どこでも同じスペーシングを使用する—リズムなしでは、レイアウトは単調に感じる

### ビジュアルディテール
**DO**: ブランドを強化する意図的で目的のある装飾要素を使用する
**DON'T**: どこでもグラスモーフィズムを使用する—ブラーエフェクト、グラスカード、グローボーダーを目的ではなく装飾的に使用
**DON'T**: 片側に太い色付きボーダーの丸い要素を使用する—意図的に見えることがほとんどない安易なアクセント
**DON'T**: 装飾としてスパークラインを使用する—洗練されて見えるが意味のあるものを伝えない小さなチャート
**DON'T**: 汎用的なドロップシャドウ付きの丸い矩形を使用する—安全で忘れられる、どのAI出力にもなりうる
**DON'T**: 本当にもっと良い代替がない限りモーダルを使用する—モーダルは安易

### モーション
→ *タイミング、イージング、モーション軽減については[モーションリファレンス](reference/motion-design.md)を参照。*

ハイインパクトな瞬間に集中する：段階的表示を伴う一つの統合されたページロードの方が、散在するマイクロインタラクションより楽しさを生む。

**DO**: 状態変化を伝えるためにモーションを使用する—登場、退出、フィードバック
**DO**: 自然な減速のために指数関数的イージング（ease-out-quart/quint/expo）を使用する
**DO**: 高さアニメーションにはheightを直接アニメーションするのではなくgrid-template-rowsトランジションを使用する
**DON'T**: レイアウトプロパティ（width、height、padding、margin）をアニメーション化する—transformとopacityのみ使用
**DON'T**: bounceやelasticイージングを使用する—時代遅れでダサい；実物はスムーズに減速する

### インタラクション
→ *フォーム、フォーカス、ローディングパターンについては[インタラクションリファレンス](reference/interaction-design.md)を参照。*

インタラクションを高速に感じさせる。楽観的UI—即座に更新し、後で同期する。

**DO**: 段階的開示を使用する—シンプルに始め、インタラクションを通じて洗練さを明らかにする（まず基本オプション、展開可能セクションの裏に高度な設定；セカンダリアクションを表示するホバー状態）
**DO**: 単に「ここには何もありません」と言うのではなく、インターフェースを教える空の状態をデザインする
**DO**: すべてのインタラクティブサーフェスを意図的でレスポンシブに感じさせる
**DON'T**: 同じ情報を繰り返す—冗長なヘッダー、見出しを繰り返すイントロ
**DON'T**: すべてのボタンをプライマリにする—ゴーストボタン、テキストリンク、セカンダリスタイルを使用する；階層が重要

### レスポンシブ
→ *モバイルファースト、流動的デザイン、コンテナクエリについては[レスポンシブリファレンス](reference/responsive-design.md)を参照。*

**DO**: コンポーネントレベルのレスポンシブ性にコンテナクエリ（@container）を使用する
**DO**: 異なるコンテキストにインターフェースを適応させる—単に縮小するのではない
**DON'T**: モバイルで重要な機能を隠す—インターフェースを適応させ、切断しない

### UXライティング
→ *ラベル、エラー、空の状態については[UXライティングリファレンス](reference/ux-writing.md)を参照。*

**DO**: すべての言葉にその存在価値を持たせる
**DON'T**: ユーザーがすでに見える情報を繰り返す

## AIスロップテスト

**重要な品質チェック**: このインターフェースを誰かに見せて「AIが作った」と言ったら、すぐに信じるだろうか？はいなら、それが問題です。

独自性のあるインターフェースは「どうやって作ったの？」と聞かせるべきで、「どのAIが作ったの？」ではありません。

上のDON'Tガイドラインをレビューしてください—それらは2024-2025年のAI生成作品の指紋です。

## 実装の原則

実装の複雑さを美的ビジョンに合わせる。マキシマリストなデザインには広範なアニメーションとエフェクトを含む精巧なコードが必要です。ミニマリストまたは洗練されたデザインには抑制、精密さ、スペーシング、タイポグラフィ、微妙なディテールへの注意深い配慮が必要です。

クリエイティブに解釈し、コンテキストに本当にデザインされたと感じる予想外の選択をしてください。同じデザインは二つと存在すべきではありません。ライトとダークのテーマ、異なるフォント、異なる美学の間で変化させてください。世代間で共通の選択に収束することは絶対にしないでください。

覚えておいてください：Claudeは並外れたクリエイティブな仕事ができます。遠慮しないでください—既成概念にとらわれず、独自のビジョンに完全にコミットすることで、真に何が作れるかを示してください。
