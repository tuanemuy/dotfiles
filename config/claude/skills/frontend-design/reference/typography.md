# タイポグラフィ

## クラシックなタイポグラフィの原則

### 垂直リズム

行の高さはすべての垂直スペーシングのベースユニットであるべきです。本文テキストが`16px`に`line-height: 1.5`（= 24px）の場合、スペーシング値は24pxの倍数であるべきです。これが無意識の調和を生み出します—テキストとスペースが数学的な基盤を共有します。

### モジュラースケールと階層

よくある間違い：近すぎる多くのフォントサイズ（14px、15px、16px、18px...）。これは曖昧な階層を作ります。

**より少ないサイズで、より多くのコントラストを。** 5サイズのシステムでほとんどのニーズをカバーできます：

| 役割 | 典型的な比率 | ユースケース |
|------|---------------|----------|
| xs | 0.75rem | キャプション、法的文言 |
| sm | 0.875rem | セカンダリUI、メタデータ |
| base | 1rem | 本文テキスト |
| lg | 1.25-1.5rem | 小見出し、リードテキスト |
| xl+ | 2-4rem | 見出し、ヒーローテキスト |

人気の比率：1.25（長三度）、1.333（完全四度）、1.5（完全五度）。一つ選んでコミットする。

### 可読性とメジャー

文字ベースのメジャーには`ch`単位を使用（`max-width: 65ch`）。行の高さは行の長さと逆比例してスケール—狭いカラムにはよりタイトなレディング、広いカラムにはより多くが必要。

**非自明なポイント**: 暗い背景の上の明るいテキストには行の高さを増やしてください。知覚されるウェイトが軽いため、テキストにはより多くの呼吸スペースが必要です。通常の行の高さに0.05-0.1を追加。

## フォント選択とペアリング

### 個性的なフォントの選択

**見えないデフォルトを避ける**: Inter、Roboto、Open Sans、Lato、Montserrat。これらはどこにでもあり、デザインを汎用的に感じさせます。個性が目標でないドキュメントやツールには問題ありませんが、独自性のあるデザインを望むなら他を探してください。

**より良いGoogle Fontsの代替**：
- Interの代わりに → **Instrument Sans**、**Plus Jakarta Sans**、**Outfit**
- Robotoの代わりに → **Onest**、**Figtree**、**Urbanist**
- Open Sansの代わりに → **Source Sans 3**、**Nunito Sans**、**DM Sans**
- エディトリアル/プレミアム感 → **Fraunces**、**Newsreader**、**Lora**

**システムフォントは過小評価されている**: `-apple-system, BlinkMacSystemFont, "Segoe UI", system-ui`はネイティブに見え、即座にロードされ、高い可読性。パフォーマンス > 個性のアプリにはこれを検討。

### ペアリングの原則

**非自明な真実**: 2番目のフォントが必要ないことが多い。一つの適切なフォントファミリーを複数のウェイトで使用すると、2つの競合するタイプフェースよりもクリーンな階層を作ります。本当のコントラストが必要な場合（例：ディスプレイ見出し＋本文セリフ）のみ2番目を追加。

ペアリングでは複数の軸でコントラストを：
- セリフ＋サンセリフ（構造のコントラスト）
- ジオメトリック＋ヒューマニスト（個性のコントラスト）
- コンデンスドディスプレイ＋ワイドボディ（プロポーションのコントラスト）

**似ているが同一でないフォントをペアリングしない**（例：2つのジオメトリックサンセリフ）。明確な階層なしにビジュアルテンションを生み出します。

### Webフォントローディング

レイアウトシフト問題：フォントが遅れてロードされ、テキストがリフローし、ユーザーがコンテンツのジャンプを見る。解決策：

```css
/* 1. 表示にfont-display: swapを使用 */
@font-face {
  font-family: 'CustomFont';
  src: url('font.woff2') format('woff2');
  font-display: swap;
}

/* 2. フォールバックメトリクスを合わせてシフトを最小化 */
@font-face {
  font-family: 'CustomFont-Fallback';
  src: local('Arial');
  size-adjust: 105%;        /* x-heightに合わせてスケール */
  ascent-override: 90%;     /* アセンダーの高さに合わせる */
  descent-override: 20%;    /* ディセンダーの深さに合わせる */
  line-gap-override: 10%;   /* 行間に合わせる */
}

body {
  font-family: 'CustomFont', 'CustomFont-Fallback', sans-serif;
}
```

[Fontaine](https://github.com/unjs/fontaine)のようなツールがこれらのオーバーライドを自動計算します。

## モダンWebタイポグラフィ

### 流動的なタイプ

流動的なタイポグラフィに`clamp(min, preferred, max)`を使用。中間値（例：`5vw + 1rem`）がスケーリング率を制御—vwが高い = スケーリングが速い。小さい画面で0に崩壊しないようremオフセットを追加。

**流動的タイプを使用しないとき**: ボタンテキスト、ラベル、UI要素（一貫性が必要）、非常に短いテキスト、または正確なブレークポイント制御が必要な場合。

### OpenType機能

ほとんどの開発者はこれらの存在を知りません。ポリッシュに使用：

```css
/* データ整列のための表形式数字 */
.data-table { font-variant-numeric: tabular-nums; }

/* 適切な分数 */
.recipe-amount { font-variant-numeric: diagonal-fractions; }

/* 略語のためのスモールキャップ */
abbr { font-variant-caps: all-small-caps; }

/* コードでの合字を無効化 */
code { font-variant-ligatures: none; }

/* カーニングを有効化（通常デフォルトでオンだが、明示的に） */
body { font-kerning: normal; }
```

フォントがサポートする機能は[Wakamai Fondue](https://wakamaifondue.com/)で確認。

## タイポグラフィシステムアーキテクチャ

値（`--font-size-16`）ではなくセマンティックにトークンを名前付け（`--text-body`、`--text-heading`）。トークンシステムにはフォントスタック、サイズスケール、ウェイト、行の高さ、字間を含める。

## アクセシビリティの考慮事項

コントラスト比（十分に文書化されている）以外に、以下を考慮：

- **ズームを無効にしない**: `user-scalable=no`はアクセシビリティを壊す。200%ズームでレイアウトが崩れるなら、レイアウトを修正する。
- **フォントサイズにrem/emを使用**: ユーザーのブラウザ設定を尊重。本文テキストに`px`は使わない。
- **最小16pxの本文テキスト**: これより小さいと目に負担がかかり、モバイルでWCAGに不合格。
- **適切なタッチターゲット**: テキストリンクには44px以上のタップターゲットを作るパディングまたは行の高さが必要。

---

**避けるべきこと**: プロジェクトごとに2-3以上のフォントファミリー。フォールバックフォント定義を省く。フォントローディングのパフォーマンス（FOUT/FOIT）を無視。本文テキストに装飾フォントを使用。
