# 空間デザイン

## スペーシングシステム

### 8ptではなく4ptベースを使用する

8ptシステムは粗すぎます—12px（8と16の間）が頻繁に必要になります。粒度のために4ptを使用：4、8、12、16、24、32、48、64、96px。

### トークンにセマンティックな名前を付ける

値（`--spacing-8`）ではなく関係性（`--space-sm`、`--space-lg`）で名前を付ける。兄弟要素のスペーシングにはマージンの代わりに`gap`を使用—マージンの崩壊やクリーンアップハックを排除。

## グリッドシステム

### 自己調整グリッド

ブレークポイントなしのレスポンシブグリッドには`repeat(auto-fit, minmax(280px, 1fr))`を使用。カラムは最低280px、行に収まるだけ多く、余りは引き伸ばし。複雑なレイアウトには名前付きグリッドエリア（`grid-template-areas`）を使用し、ブレークポイントで再定義。

## ビジュアル階層

### スクイントテスト

目をぼかす（またはスクリーンショットを撮ってブラー）。以下を識別できるか：
- 最も重要な要素？
- 2番目に重要な要素？
- 明確なグルーピング？

ぼかした状態ですべてが同じ重みに見えたら、階層に問題があります。

### 複数の次元による階層

サイズだけに頼らないでください。組み合わせる：

| ツール | 強い階層 | 弱い階層 |
|------|------------------|----------------|
| **サイズ** | 3:1以上の比率 | 2:1未満の比率 |
| **ウェイト** | 太字 vs レギュラー | ミディアム vs レギュラー |
| **色** | ハイコントラスト | 似たトーン |
| **位置** | 上/左（プライマリ） | 下/右 |
| **スペース** | ホワイトスペースに囲まれている | 詰まっている |

**最良の階層は2-3次元を同時に使用**: より大きく、より太く、かつその上にスペースが多い見出し。

### カードは必須ではない

カードは使われすぎています。スペーシングとアラインメントが自然にビジュアルグルーピングを作ります。カードを使用するのは、コンテンツが本当に独立していてアクションが可能な場合、アイテムがグリッドでの視覚的な比較を必要とする場合、またはコンテンツに明確なインタラクション境界が必要な場合のみ。**カードの中にカードをネストしない**—カード内の階層にはスペーシング、タイポグラフィ、微妙なディバイダーを使用する。

## コンテナクエリ

ビューポートクエリはページレイアウト用。**コンテナクエリはコンポーネント用**：

```css
.card-container {
  container-type: inline-size;
}

.card {
  display: grid;
  gap: var(--space-md);
}

/* カードのレイアウトはビューポートではなくコンテナに基づいて変化 */
@container (min-width: 400px) {
  .card {
    grid-template-columns: 120px 1fr;
  }
}
```

**なぜ重要か**: 狭いサイドバーのカードはコンパクトなまま、同じカードがメインコンテンツエリアでは展開される—ビューポートハックなしで自動的に。

## 光学的調整

`margin-left: 0`のテキストはレターフォームのホワイトスペースにより字下げして見える—光学的に揃えるために負のマージン（`-0.05em`）を使用。幾何学的に中心に配置されたアイコンはしばしばずれて見える；再生アイコンは右にシフト、矢印はその方向にシフトが必要。

### タッチターゲット vs ビジュアルサイズ

ボタンは小さく見えても大きなタッチターゲット（最小44px）が必要。パディングまたは疑似要素を使用：

```css
.icon-button {
  width: 24px;  /* ビジュアルサイズ */
  height: 24px;
  position: relative;
}

.icon-button::before {
  content: '';
  position: absolute;
  inset: -10px;  /* タップターゲットを44pxに拡張 */
}
```

## 奥行きとエレベーション

任意の数値ではなくセマンティックなz-indexスケール（dropdown → sticky → modal-backdrop → modal → toast → tooltip）を作成。シャドウには一貫したエレベーションスケール（sm → md → lg → xl）を作成。**重要なポイント**: シャドウは微妙であるべき—はっきり見えるなら、おそらく強すぎます。

---

**避けるべきこと**: スケール外の任意のスペーシング値。すべてのスペーシングを等しくすること（変化が階層を作る）。サイズだけで階層を作ること—サイズ、ウェイト、色、スペースを組み合わせる。
