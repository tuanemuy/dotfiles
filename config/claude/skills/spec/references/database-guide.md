# Phase 5: DB設計ガイド

各ドメインのエンティティに基づいてデータベース設計を行う。

出力先: `spec/database/index.md`

各テーブルについて以下を記載する:
- **テーブル名**
- **カラム一覧**: カラム名、型、制約（NOT NULL, UNIQUE 等）
- **主キー・外部キー**
- **インデックス**（必要な場合）
- **リレーション**: 他テーブルとの関係

## レビュー

出力先: `spec/database/review/${連番}.md`

レビュー観点:
- すべてのエンティティ・値オブジェクトの永続化先が定義されているか
- カラムの型・制約がドメインモデルのフィールド定義と整合しているか
- 適切に正規化されているか
- パフォーマンスに問題がないか
- データ整合性が保証されているか
- ドメインごとに必要なファイルのみ読み込みながらレビューする

問題があれば修正し、再レビュー。再レビュー時は前回のレビューファイルを参照せず、チェックリストに沿って一から評価する。クリーンになるまで繰り返す。
