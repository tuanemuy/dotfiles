# テスト実装ガイド

`spec/testcases/` の定義に基づいて、テストを実装する。

## サブエージェントへの指示テンプレート

ドメインごとにサブエージェントを起動する。

```
あなたは設計ドキュメントに基づいてテストを実装するエンジニアです。

## 前提
- まずプロジェクトルートの CLAUDE.md を読み、使用言語・ディレクトリ構成・規約・コマンドを把握する

## 対象
- ドメイン: {domain名}
- テストケース定義: spec/testcases/{domain}/
- ユースケース設計: spec/usecases/{domain}.md

## やること
1. spec/testcases/{domain}/ のテストケース定義を読む
2. 対応する実装コードを Read で読む
3. テストケース定義に従ってテストを実装する
4. CLAUDE.md に記載の typecheck・lint コマンドを実行し、エラーがなくなるまで修正する

## 重要な原則
- 実装に合わせず、仕様に合わせたテストを作成する
- テストケース定義のカバレッジを網羅する
- やむを得ずリンタールールに違反する場合は、直前の行にコメントで理由を追記する
- 既存のテストパターンに合わせる

## 出力
- 作成したテストファイルの一覧
- typecheck・lint の実行結果
- 設計との乖離（あれば: 乖離の内容・理由・実装での対応方針）
- テストケース定義でカバーできなかった項目（あれば理由付きで）
```

## 完了確認

サブエージェント完了後、メインエージェントが以下を実行する:
1. typecheck + lint
2. **テスト実行**（CLAUDE.md のテストコマンドに従う）
3. 失敗するテストがあればサブエージェントを再起動して修正させる（最大3回）
