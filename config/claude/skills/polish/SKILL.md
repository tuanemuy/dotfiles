**最初に**: デザイン原則とアンチパターンについてfrontend-designスキルを使用してください。

## 視覚的な評価

HTMLファイルを対象とする場合、スクリーンショットを撮影してアラインメント・スペーシング・一貫性を視覚的に確認する。

```bash
bash <skill-path>/../critique/scripts/screenshot.sh /path/to/target.html /tmp/polish-screenshots
```

`Read` ツールで各画像を読み込んでから作業を開始する。

良い仕事と素晴らしい仕事を分けるすべての小さなディテールをキャッチするための綿密な最終パスを実行します。出荷されたものとポリッシュされたものの違いです。

## ポリッシュ前の評価

現状と目標を理解する：

1. **完了度をレビュー**：
   - 機能的に完了しているか？
   - 保持すべき既知の問題があるか（TODOでマーク）？
   - 品質基準は？（MVP vs フラッグシップ機能？）
   - いつ出荷するか？（ポリッシュにどれだけ時間があるか？）

2. **ポリッシュ領域を特定する**：
   - ビジュアルの不整合
   - スペーシングとアラインメントの問題
   - インタラクション状態のギャップ
   - コピーの不整合
   - エッジケースとエラー状態
   - ローディングとトランジションのスムーズさ

**重要**: ポリッシュは最後のステップであり、最初ではありません。機能的に完了していない作業をポリッシュしないでください。

## 体系的にポリッシュする

これらの次元を方法的に作業する：

### ビジュアルアラインメントとスペーシング

- **ピクセルパーフェクトなアラインメント**: すべてがグリッドに揃う
- **一貫したスペーシング**: すべてのギャップがスペーシングスケールを使用（ランダムな13pxギャップなし）
- **光学的アラインメント**: 視覚的な重みに合わせて調整（アイコンは光学的な中心のためにオフセットが必要な場合がある）
- **レスポンシブの一貫性**: スペーシングとアラインメントがすべてのブレークポイントで機能する
- **グリッドの遵守**: 要素がベースライングリッドにスナップする

**チェック**：
- グリッドオーバーレイを有効にしてアラインメントを検証
- ブラウザインスペクターでスペーシングを確認
- 複数のビューポートサイズでテスト
- 「違和感がある」要素を探す

### タイポグラフィの洗練

- **階層の一貫性**: 同じ要素が全体を通じて同じサイズ/ウェイトを使用
- **行の長さ**: 本文テキストで45-75文字
- **行の高さ**: フォントサイズとコンテキストに適切
- **ウィドウとオーファン**: 最終行に単一の単語なし
- **ハイフネーション**: 言語とカラム幅に適切
- **カーニング**: 必要に応じて字間を調整（特に見出し）
- **フォントローディング**: FOUT/FOITフラッシュなし

### 色とコントラスト

- **コントラスト比**: すべてのテキストがWCAG基準を満たす
- **一貫したトークン使用**: ハードコードされた色なし、すべてデザイントークンを使用
- **テーマの一貫性**: すべてのテーマバリアントで機能する
- **色の意味**: 同じ色が全体で同じことを意味する
- **アクセシブルなフォーカス**: 十分なコントラストで見えるフォーカスインジケーター
- **ティンテッドニュートラル**: ピュアグレーやピュアブラックなし—微妙な色ティント（0.01クロマ）を追加
- **色の上のグレー**: 色付き背景にグレーテキストを置かない—その色の色合いまたは透明度を使用

### インタラクション状態

すべてのインタラクティブ要素にすべての状態が必要：

- **デフォルト**: 休止状態
- **ホバー**: 微妙なフィードバック（色、スケール、シャドウ）
- **フォーカス**: キーボードフォーカスインジケーター（代替なしに削除しない）
- **アクティブ**: クリック/タップフィードバック
- **無効**: 明確に非インタラクティブ
- **ローディング**: 非同期アクションのフィードバック
- **エラー**: バリデーションまたはエラー状態
- **成功**: 正常完了

**欠けている状態は混乱と壊れた体験を生みます**。

### マイクロインタラクションとトランジション

- **スムーズなトランジション**: すべての状態変化が適切にアニメーション化（150-300ms）
- **一貫したイージング**: 自然な減速のためにease-out-quart/quint/expoを使用。bounceやelasticは絶対に使わない—時代遅れに感じる。
- **カクつきなし**: 60fpsアニメーション、transformとopacityのみアニメーション
- **適切なモーション**: モーションは装飾ではなく目的に奉仕する
- **モーション軽減**: `prefers-reduced-motion`を尊重する

### コンテンツとコピー

- **一貫した用語**: 同じものが全体で同じ名前で呼ばれる
- **一貫した大文字**: タイトルケース vs センテンスケースが一貫して適用される
- **文法とスペリング**: タイプミスなし
- **適切な長さ**: 冗長すぎず、簡潔すぎない
- **句読点の一貫性**: 文にはピリオド、ラベルにはなし（すべてのラベルにある場合を除く）

### アイコンと画像

- **一貫したスタイル**: すべてのアイコンが同じファミリーまたは合ったスタイル
- **適切なサイジング**: アイコンがコンテキストに対して一貫したサイズ
- **適切なアラインメント**: アイコンが隣接するテキストに光学的に揃う
- **altテキスト**: すべての画像に説明的なaltテキスト
- **ローディング状態**: 画像がレイアウトシフトを引き起こさない、適切なアスペクト比
- **Retinaサポート**: 高DPIスクリーン用の2xアセット

### フォームと入力

- **ラベルの一貫性**: すべての入力に適切なラベル
- **必須インジケーター**: 明確で一貫
- **エラーメッセージ**: 有用で一貫
- **タブ順序**: 論理的なキーボードナビゲーション
- **オートフォーカス**: 適切（使いすぎない）
- **バリデーションタイミング**: 一貫（onblur vs onsubmit）

### エッジケースとエラー状態

- **ローディング状態**: すべての非同期アクションにローディングフィードバック
- **空の状態**: 空白スペースではなく有用な空の状態
- **エラー状態**: リカバリーパス付きの明確なエラーメッセージ
- **成功状態**: 成功アクションの確認
- **長いコンテンツ**: 非常に長い名前、説明などを処理
- **コンテンツなし**: 欠落データを適切に処理
- **オフライン**: 適切なオフラインハンドリング（該当する場合）

### レスポンシブ

- **すべてのブレークポイント**: モバイル、タブレット、デスクトップをテスト
- **タッチターゲット**: タッチデバイスで最小44x44px
- **読みやすいテキスト**: モバイルで14px未満のテキストなし
- **水平スクロールなし**: コンテンツがビューポートに収まる
- **適切なリフロー**: コンテンツが論理的に適応する

### パフォーマンス

- **高速な初期読み込み**: クリティカルパスの最適化
- **レイアウトシフトなし**: ロード後に要素がジャンプしない（CLS）
- **スムーズなインタラクション**: ラグやカクつきなし
- **最適化された画像**: 適切なフォーマットとサイズ
- **遅延読み込み**: 画面外のコンテンツを遅延読み込み

### コード品質

- **console.logの削除**: プロダクションにデバッグログなし
- **コメントアウトされたコードの削除**: デッドコードのクリーンアップ
- **未使用インポートの削除**: 未使用の依存関係のクリーンアップ
- **一貫した命名**: 変数と関数が規約に従う
- **型安全性**: TypeScriptの`any`や無視されたエラーなし
- **アクセシビリティ**: 適切なARIAラベルとセマンティックHTML

## ポリッシュチェックリスト

体系的に確認する：

- [ ] すべてのブレークポイントでビジュアルアラインメントが完璧
- [ ] スペーシングがデザイントークンを一貫して使用
- [ ] タイポグラフィ階層が一貫
- [ ] すべてのインタラクション状態が実装済み
- [ ] すべてのトランジションがスムーズ（60fps）
- [ ] コピーが一貫してポリッシュ済み
- [ ] アイコンが一貫して適切なサイズ
- [ ] すべてのフォームに適切なラベルとバリデーション
- [ ] エラー状態が有用
- [ ] ローディング状態が明確
- [ ] 空の状態が歓迎的
- [ ] タッチターゲットが最小44x44px
- [ ] コントラスト比がWCAG AAを満たす
- [ ] キーボードナビゲーションが機能
- [ ] フォーカスインジケーターが見える
- [ ] コンソールエラーや警告なし
- [ ] ロード時のレイアウトシフトなし
- [ ] サポートするすべてのブラウザで動作
- [ ] モーション軽減プリファレンスを尊重
- [ ] コードがクリーン（TODO、console.log、コメントアウトされたコードなし）

**重要**: ポリッシュはディテールについてです。ズームインする。目を細める。自分で使う。小さなことが積み重なります。

**絶対にしてはいけないこと**：
- 機能的に完了する前にポリッシュする
- 30分で出荷する場合にポリッシュに何時間もかける（トリアージする）
- ポリッシュ中にバグを導入する（徹底的にテスト）
- 体系的な問題を無視する（スペーシングがどこでもずれている場合、システムを修正する）
- 他を粗いまま残して一つのものを完璧にする（一貫した品質レベル）

## 最終検証

完了マークの前に：

- **自分で使う**: 実際に機能とインタラクションする
- **実際のデバイスでテスト**: ブラウザDevToolsだけでなく
- **他の人にレビューを依頼**: フレッシュな目は見落としをキャッチする
- **デザインと比較**: 意図したデザインに一致
- **すべての状態をチェック**: ハッピーパスだけでなくテスト

覚えておいてください：あなたは完璧な注意力と絶妙な感性を持っています。努力なしに感じ、意図的に見え、完璧に機能するまでポリッシュしてください。ディテールにこだわってください—それは重要です。
