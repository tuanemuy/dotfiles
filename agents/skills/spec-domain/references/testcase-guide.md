# Phase 6: テストケース定義ガイド

すべてのユースケースについて、Markdownテーブル形式でテストケースを定義する。

出力先: `spec/testcases/${domain}/${usecase}.md`

テーブル構成:

```markdown
| 前提条件 | 操作 | 期待結果 | 実装ステータス |
|---|---|---|---|
| テストの前提状態 | 実行する操作 | 期待される振る舞い | （空欄 — 実装時に埋める） |
```

以下を網羅する:
- **正常系**: ユースケースの基本フロー
- **異常系**: バリデーションエラー、ビジネスルール違反、外部サービスエラー
- **境界値**: エンティティの不変条件の境界
- **エッジケース**: シナリオの異常系フロー

ユースケース設計で定義したエラーケースがテストケースでカバーされていることを確認する。
