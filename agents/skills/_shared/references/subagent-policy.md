# サブエージェント委譲ポリシー

スキル群で「サブエージェント」と呼ぶ委譲先は、以下の優先順位で実行する。

## 実行方式の優先順位

1. **実行環境がサブエージェントをサポートする場合**（例: Claude Code の Task 系、Anthropic Agent SDK のマルチエージェント機能など）
   → 各作業を**別々のサブエージェント**として起動する。
2. **実行環境がプロセスレベルのエージェント実行をサポートする場合**（OSプロセス単位でエージェントを起動できる場合）
   → 各作業を**独立したOSプロセス**として起動する。
3. **上記いずれも利用できない場合のみ**
   → 親エージェント内で各作業を順に実行する（シミュレート）。

サブエージェントもプロセスランナーも利用できる環境で、親エージェント内に全作業を閉じ込めて逐次シミュレートしてはいけない。

「サブエージェントが許可されている場合は」のような条件分岐は不要。サブエージェントまたはプロセスが使えるなら必ずそちらを使う。

## 並列実行の原則

- 独立した作業は**必ず並列で**起動する。直列だと遅すぎる。
- 依存関係がある場合のみ順序を守る（依存先を先に完了させる）。

## 委譲時のコンテキスト

- 各委譲先には**独立したコンテキスト**を渡す。前の結果を前提にしない（ゼロベースで実行させる）。
- 委譲先には必ず**プロジェクトルートの CLAUDE.md を最初に読むよう指示する** — コマンド・規約・構成の情報源として。
- 複数の委譲先に同じプロンプトを渡すと似た結論に収束しやすい。視点を分ける場合は視点ごとに明確に指示を変える。

## 失敗時のフォールバック

- いずれかの委譲先が失敗・タイムアウトした場合は、成功した結果だけで柔軟に進める。
- 全部失敗した場合のみ親エージェントでやり直す。

## 参照方法

各スキルは本ドキュメントを参照することで、委譲方式の判断と並列実行の原則を共通化する。個別スキルでは「どの作業を委譲するか」「委譲先に何を渡すか」だけを記述し、実行方式の判定や並列化の原則はここに集約する。
