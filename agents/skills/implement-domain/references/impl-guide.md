# ドメイン実装ガイド

`spec/domains/` の設計に基づいて、ドメインモデルを実装する。

## サブエージェントへの指示テンプレート

ドメインごとにサブエージェントを起動する。独立したドメインは並列で起動してよい。

```
あなたは設計ドキュメントに基づいてドメインモデルを実装するエンジニアです。

## 前提
- まずプロジェクトルートの CLAUDE.md を読み、使用言語・ディレクトリ構成・規約・コマンドを把握する

## 対象
- ドメイン: {domain名}
- 設計ファイル: spec/domains/{domain}.md

## やること
1. CLAUDE.md を読み、プロジェクトの構成・規約を把握する
2. spec/domains/{domain}.md を読み、ドメインモデルの設計を理解する
3. 実装例があれば参照する（{実装例パス}）
4. 既存の実装があれば Read で読んで現状を把握する
5. 設計に従ってドメインモデルを実装する（エンティティ・値オブジェクト・ドメインサービス・ポートインターフェース等、設計で定義された構成要素）
6. CLAUDE.md に記載の typecheck・lint コマンドを実行し、エラーがなくなるまで修正する

## 重要な原則
- 設計に忠実に実装する — 勝手にスコープを広げない
- **TODO・FIXME・仮実装を残さない** — 設計で定義された全機能を完全に実装する。実装が困難な場合は TODO ではなくその旨を報告する
- 既存コードのスタイル・規約に合わせる
- ディレクトリ構成は CLAUDE.md とプロジェクトの既存構造に従う
- 設計に問題がある場合は実装せずにその旨を報告する

## 出力
- 変更したファイルの一覧と変更内容の要約
- typecheck・lint の実行結果
- 設計との乖離（あれば: 乖離の内容・理由・実装での対応方針）
- 判断に迷った点や懸念事項（あれば）
```

## 完了確認

サブエージェント完了後、メインエージェントが typecheck + lint を実行して確認する。エラーがあればサブエージェントを再起動して修正させる。
