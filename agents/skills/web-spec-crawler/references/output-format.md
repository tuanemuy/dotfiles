# spec/ 出力フォーマット（網羅版）

frontier ドレイン完了後の最終出力。基本構成は web-spec-explorer と同じだが、網羅特化として次を追加・強化する:

- 各ページに**全操作要素テーブル**（主要アクションだけでなく全要素）
- 各ページに**全分岐テーブル**（状態変化・条件表示・フォーム結果を漏れなく）
- **完全な画面遷移図**（back-edge 含む）
- **カバレッジサマリ**（探索済み/除外の集計＝網羅の証明）

web-spec-explorer の `references/output-format.md`（overview / sitemap / auth / components / flows のテンプレート）はそのまま流用できる。本ドキュメントは網羅版で**変わる/増える部分**だけ示す。

## ディレクトリ構成

```
spec/
├── overview.md          # サービス概要（web-spec-explorer と同じ＋カバレッジサマリへの参照）
├── sitemap.md           # 画面一覧・サイト構造・完全な遷移図
├── auth.md              # 認証・認可（ロール別アクセス表を網羅）
├── components.md        # 共通UIコンポーネント
├── pages/
│   └── {page-name}.md   # 各画面（全操作要素・全分岐を網羅）
├── flows/
│   └── {flow-name}.md   # 主要フロー（web-spec-explorer と同じ）
└── _coverage/
    ├── strategy.md      # 合意した戦略
    ├── pages.md         # ページ frontier（最終状態）
    ├── branches.md      # 分岐 frontier（最終状態）
    └── summary.md       # カバレッジサマリ（網羅の証明）
```

## pages/{page-name}.md（網羅版）

web-spec-explorer のページテンプレートに対し、「アクション」を**全操作要素**に拡張し、「状態・表示切替」を**全分岐**に拡張する。一部抜粋ではなく、台帳の当該ページの全エントリを反映する。

```markdown
# {画面名}

- URL: {url テンプレート}（代表インスタンス: {例}）
- 認証: {必要 / 不要}
- 確認ロール: {未認証 / 一般 / 管理者 など、確認したロール}
- セクション: {所属セクション}
- カバレッジ: verified（auditor 裏取り済み） / excluded（理由）

## 目的

{この画面が何のためにあるか}

## レイアウト

{画面の大まかな構成}

## 全操作要素

このページの操作可能な要素を漏れなく列挙する（主要なものだけにしない）。

| ラベル | 要素種別 | 分類 | 振る舞い・遷移先/開く内容 | 状態（disabled等） |
|--------|---------|------|------------------------|------------------|
| ログイン | ボタン(primary) | フォーム送信 | → /dashboard（成功時） | 入力不完全で disabled |
| パスワードを忘れた | リンク | ナビゲーション | → /forgot-password | - |
| 表示切替 | トグル | 状態変化 | パスワードの表示/非表示 | - |

## 全分岐

状態変化・条件表示・フォーム結果を漏れなく。台帳 branches.md の当該ページ分を反映。

| 分岐 | トリガー | 条件 | 結果・現れる要素 | さらなる分岐 |
|------|---------|------|----------------|------------|
| ログイン失敗 | 「ログイン」送信 | 認証情報不一致 | エラー「メールまたはパスワードが正しくありません」 | - |
| 空送信 | 「ログイン」送信 | 各欄が空 | 「必須です」フィールド下に表示 | - |
| 設定モーダル | 「設定」クリック | - | 設定モーダル展開（保存/キャンセル/各項目） | → 設定保存フロー |

## sitemap.md（遷移図を完全化）

web-spec-explorer の sitemap に加え、**到達した全遷移**を図にする。back-edge（既知状態へ戻る遷移）も含めることで「網羅した遷移グラフ」になる。

```markdown
## 画面遷移図（完全版）

到達確認した全遷移。実線=新規ページへの遷移、点線=既知ページへの戻り(back-edge)。

\`\`\`mermaid
graph TD
  P01[トップ /] --> P02[ログイン /login]
  P01 --> P06[料金 /pricing]
  P02 -->|送信成功| P03[ダッシュボード /dashboard]
  P03 --> P04[アイテム詳細 /items/:id]
  P04 -.戻る.-> P03
  P03 -->|設定モーダル| P03
\`\`\`
```

mermaid が使えない環境ではインデントツリー＋遷移リストで代替する。

## _coverage/summary.md（網羅の証明）

`coverage-ledger.md` のカバレッジサマリをそのまま最終成果物とする。`discovered` 残り 0 を明記することが「完走した」という主張の根拠になる。

## 構造化のポイント

- **台帳が一次情報、spec/ は読み物**: pages/ や sitemap は台帳（_coverage/）から起こす。台帳と spec/ の数が食い違ったら台帳が正。最後に突き合わせる
- **重複排除と共通コンポーネント抽出**: 複数ページに出る UI は components.md に集約し、各ページからは参照する（全ページに同じ内容を書き写さない）
- **excluded を隠さない**: 探索しなかった箇所は summary.md と該当ページに理由つきで明示する。「網羅した」と書きつつ黙ってスキップするのは虚偽になる
- **要確認を残す**: auditor が「要確認」とした動的要素などは「TODO: 要確認」として残し、ユーザーレビュー（Phase 7）に回す
```
