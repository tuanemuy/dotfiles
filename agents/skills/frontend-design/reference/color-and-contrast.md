# 色とコントラスト

## カラースペース：OKLCHを使用する

**HSLの使用をやめてください。** 代わりにOKLCH（またはLCH）を使用してください。知覚的に均一で、明度の等しいステップが等しく*見えます*—HSLでは黄色の50%明度は明るく見えますが、青の50%は暗く見えます。

```css
/* OKLCH: 明度 (0-100%), 彩度 (0-0.4+), 色相 (0-360) */
--color-primary: oklch(60% 0.15 250);      /* 青 */
--color-primary-light: oklch(85% 0.08 250); /* 同じ色相、より明るい */
--color-primary-dark: oklch(35% 0.12 250);  /* 同じ色相、より暗い */
```

**重要なポイント**: 白や黒に近づくにつれて、彩度（サチュレーション）を下げてください。極端な明度での高彩度はけばけばしく見えます。85%明度のライトブルーには約0.08の彩度が必要で、ベースカラーの0.15ではありません。

## 機能的なパレットの構築

### ティンテッドニュートラルの罠

**ピュアグレーは死んだ。** すべてのニュートラルにブランドの色相をわずかに加えてください：

```css
/* 死んだグレー */
--gray-100: oklch(95% 0 0);     /* 個性がない */
--gray-900: oklch(15% 0 0);

/* ウォームティンテッドグレー（ブランドの温かみを追加） */
--gray-100: oklch(95% 0.01 60);  /* わずかな温かみ */
--gray-900: oklch(15% 0.01 60);

/* クールティンテッドグレー（テック、プロフェッショナル） */
--gray-100: oklch(95% 0.01 250); /* わずかな青み */
--gray-900: oklch(15% 0.01 250);
```

彩度はごくわずか（0.01）ですが知覚可能です。ブランドカラーとUIの間に無意識の一貫性を生み出します。

### パレット構造

完全なシステムには以下が必要：

| 役割 | 目的 | 例 |
|------|---------|---------|
| **プライマリ** | ブランド、CTA、主要アクション | 1色、3-5段階のシェード |
| **ニュートラル** | テキスト、背景、ボーダー | 9-11段階のスケール |
| **セマンティック** | 成功、エラー、警告、情報 | 4色、各2-3段階のシェード |
| **サーフェス** | カード、モーダル、オーバーレイ | 2-3段階のエレベーション |

**必要でない限りセカンダリ/ターシャリはスキップ。** ほとんどのアプリはアクセントカラー1つで十分です。追加すると判断疲れとビジュアルノイズが生まれます。

### 60-30-10ルール（正しい適用方法）

このルールは**ビジュアルウェイト**についてであり、ピクセル数ではありません：

- **60%**: ニュートラルな背景、ホワイトスペース、ベースサーフェス
- **30%**: セカンダリカラー—テキスト、ボーダー、非アクティブ状態
- **10%**: アクセント—CTA、ハイライト、フォーカス状態

よくある間違い：「ブランドカラーだから」とアクセントカラーをどこでも使用すること。アクセントカラーは*稀だからこそ*機能します。使いすぎるとその力が失われます。

## コントラストとアクセシビリティ

### WCAG要件

| コンテンツタイプ | AA最低基準 | AAA目標 |
|--------------|------------|------------|
| 本文テキスト | 4.5:1 | 7:1 |
| 大きなテキスト（18px以上または14px太字） | 3:1 | 4.5:1 |
| UIコンポーネント、アイコン | 3:1 | 4.5:1 |
| 重要でない装飾 | なし | なし |

**注意点**: プレースホルダーテキストも4.5:1が必要です。あちこちで見かける薄いグレーのプレースホルダーは、通常WCAGに不合格です。

### 危険な色の組み合わせ

これらはコントラスト不合格または可読性の問題を引き起こすことが多い：

- 白の上の薄いグレーテキスト（アクセシビリティ不合格の第1位）
- **色付き背景の上のグレーテキスト**—グレーは色の上で洗い流されて死んで見える。背景色のより暗い色合い、または透明度を使用する
- 緑の上の赤テキスト（またはその逆）—男性の8%がこれらを区別できない
- 赤の上の青テキスト（視覚的に振動する）
- 白の上の黄色テキスト（ほぼ常に不合格）
- 画像の上の薄い細字テキスト（予測不能なコントラスト）

### ピュアグレーやピュアブラックを決して使用しない

ピュアグレー（`oklch(50% 0 0)`）とピュアブラック（`#000`）は自然に存在しません—実際の影やサーフェスには常にカラーキャストがあります。0.005-0.01の彩度でも、明らかにティンテッドに見えることなく自然に感じるのに十分です。（上のティンテッドニュートラルの例を参照。）

### テスト

目を信用しないでください。ツールを使用：

- [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
- ブラウザDevTools → レンダリング → 視覚障害のエミュレーション
- [Polypane](https://polypane.app/) リアルタイムテスト用

## テーマ：ライト＆ダークモード

### ダークモードはライトモードの反転ではない

色を単に入れ替えることはできません。ダークモードには異なるデザイン判断が必要です：

| ライトモード | ダークモード |
|------------|-----------|
| 深さにシャドウ | 深さにより明るいサーフェス（シャドウなし） |
| 明るい上の暗いテキスト | 暗い上の明るいテキスト（フォントウェイトを減らす） |
| 鮮やかなアクセント | アクセントの彩度をやや下げる |
| 白い背景 | ピュアブラックは使わない—ダークグレー（oklch 12-18%）を使用 |

```css
/* ダークモードの深さはシャドウではなくサーフェスカラーで表現 */
:root[data-theme="dark"] {
  --surface-1: oklch(15% 0.01 250);
  --surface-2: oklch(20% 0.01 250);  /* 「上」= より明るい */
  --surface-3: oklch(25% 0.01 250);

  /* テキストウェイトをやや減らす */
  --body-weight: 350;  /* 400の代わりに */
}
```

### トークン階層

2層を使用：プリミティブトークン（`--blue-500`）とセマンティックトークン（`--color-primary: var(--blue-500)`）。ダークモードではセマンティック層のみを再定義—プリミティブはそのまま。

## アルファはデザインの臭い

透明度（rgba、hsla）の多用は通常、不完全なパレットを意味します。アルファは予測不能なコントラスト、パフォーマンスオーバーヘッド、不整合を生みます。代わりに各コンテキストに明示的なオーバーレイカラーを定義してください。例外：フォーカスリングや透過が必要なインタラクティブ状態。

---

**避けるべきこと**: 情報伝達を色だけに頼ること。各色に明確な役割のないパレットを作成すること。大きな領域にピュアブラック（#000）を使用すること。色覚異常テストを省くこと（男性の8%が影響を受ける）。
