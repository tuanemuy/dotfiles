再利用可能なパターン、コンポーネント、デザイントークンを特定し、体系的な再利用のためにデザインシステムに抽出・統合します。

## 発見

ターゲットエリアを分析して抽出の機会を特定する：

1. **デザインシステムを見つける**: デザインシステム、コンポーネントライブラリ、共有UIディレクトリを探す（「design system」「ui」「components」などでgrep）。以下を理解するまで徹底的に調査する：
   - コンポーネントの組織と命名規則
   - デザイントークン構造（もしあれば）
   - ドキュメントパターン
   - インポート/エクスポート規約

   **重要**: デザインシステムが存在しない場合は、作成する前に確認してください。好ましい場所と構造を最初に理解してください。

2. **パターンを特定する**: 以下を探す：
   - **繰り返しコンポーネント**: 複数回使用される類似のUIパターン（ボタン、カード、入力など）
   - **ハードコードされた値**: トークンにすべき色、スペーシング、タイポグラフィ、シャドウ
   - **一貫性のないバリエーション**: 同じコンセプトの複数の実装（3つの異なるボタンスタイル）
   - **再利用可能なパターン**: 体系化する価値のあるレイアウトパターン、コンポジションパターン、インタラクションパターン

3. **価値を評価する**: すべてを抽出すべきではない。以下を考慮する：
   - 3回以上使用されている、または再利用される可能性が高いか？
   - 体系化することで一貫性が向上するか？
   - これは汎用的なパターンかコンテキスト固有か？
   - メンテナンスコスト vs メリットは？

## 抽出の計画

体系的な抽出計画を作成する：

- **抽出するコンポーネント**: どのUI要素が再利用可能なコンポーネントになるか？
- **作成するトークン**: どのハードコードされた値がデザイントークンになるか？
- **サポートするバリアント**: 各コンポーネントにどのバリエーションが必要か？
- **命名規則**: 既存パターンに合ったコンポーネント名、トークン名、プロップ名
- **移行パス**: 既存の使用箇所を新しい共有バージョンに変更する方法

**重要**: デザインシステムは段階的に成長します。明確に再利用可能なものを今抽出し、いつか再利用可能かもしれないものすべてを抽出するのではありません。

## 抽出と充実

改善された再利用可能なバージョンを構築する：

- **コンポーネント**: 以下を備えた設計の良いコンポーネントを作成：
  - 適切なデフォルト値を持つ明確なprops API
  - 異なるユースケースのための適切なバリアント
  - 組み込みのアクセシビリティ（ARIA、キーボードナビゲーション、フォーカス管理）
  - ドキュメントと使用例

- **デザイントークン**: 以下を備えたトークンを作成：
  - 明確な命名（プリミティブ vs セマンティック）
  - 適切な階層と組織
  - 各トークンの使用タイミングのドキュメント

- **パターン**: 以下を含むパターンの文書化：
  - このパターンを使用するタイミング
  - コード例
  - バリエーションと組み合わせ

**絶対にしてはいけないこと**：
- 汎化せずに一回限りのコンテキスト固有実装を抽出する
- 汎用的すぎて役に立たないコンポーネントを作成する
- 既存のデザインシステムの規約を考慮せずに抽出する
- 適切なTypeScript型やpropドキュメントを省略する
- すべての値にトークンを作成する（トークンにはセマンティックな意味があるべき）

## 移行

既存の使用箇所を新しい共有バージョンに置き換える：

- **すべてのインスタンスを見つける**: 抽出したパターンを検索する
- **体系的に置き換える**: 各使用箇所を共有バージョンに更新する
- **徹底的にテスト**: ビジュアルと機能の同等性を確保する
- **デッドコードを削除**: 古い実装を削除する

## 文書化

デザインシステムのドキュメントを更新する：

- 新しいコンポーネントをコンポーネントライブラリに追加する
- トークンの使用方法と値を文書化する
- 例とガイドラインを追加する
- Storybookやコンポーネントカタログを更新する

覚えておいてください：良いデザインシステムは生きたシステムです。パターンが現れたら抽出し、思慮深く充実させ、一貫してメンテナンスしてください。
