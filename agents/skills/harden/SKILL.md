理想化されたデザインを壊すエッジケース、エラー、国際化の問題、実世界の使用シナリオに対してインターフェースを強化します。

## 堅牢化のニーズの評価

弱点とエッジケースを特定する：

1. **極端な入力でテスト**：
   - 非常に長いテキスト（名前、説明、タイトル）
   - 非常に短いテキスト（空、1文字）
   - 特殊文字（絵文字、RTLテキスト、アクセント）
   - 大きな数値（百万、十億）
   - 多数のアイテム（1000以上のリスト項目、50以上のオプション）
   - データなし（空の状態）

2. **エラーシナリオをテスト**：
   - ネットワーク障害（オフライン、低速、タイムアウト）
   - APIエラー（400、401、403、404、500）
   - バリデーションエラー
   - 権限エラー
   - レート制限
   - 同時操作

3. **国際化をテスト**：
   - 長い翻訳（ドイツ語は英語より30%長いことが多い）
   - RTL言語（アラビア語、ヘブライ語）
   - 文字セット（中国語、日本語、韓国語、絵文字）
   - 日付/時刻形式
   - 数値形式（1,000 vs 1.000）
   - 通貨記号

**重要**: 完璧なデータでのみ動作するデザインはプロダクション対応ではありません。現実に対して堅牢化してください。

## 堅牢化の次元

体系的に耐障害性を改善する：

### テキストのオーバーフローと折り返し

**長いテキストの処理**：
```css
/* 省略記号付き1行 */
.truncate {
  overflow: hidden;
  text-overflow: ellipsis;
  white-space: nowrap;
}

/* クランプ付き複数行 */
.line-clamp {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}

/* 折り返しを許可 */
.wrap {
  word-wrap: break-word;
  overflow-wrap: break-word;
  hyphens: auto;
}
```

**Flex/Gridのオーバーフロー**：
```css
/* flexアイテムのオーバーフロー防止 */
.flex-item {
  min-width: 0; /* コンテンツサイズ以下の縮小を許可 */
  overflow: hidden;
}

/* gridアイテムのオーバーフロー防止 */
.grid-item {
  min-width: 0;
  min-height: 0;
}
```

**レスポンシブなテキストサイジング**：
- 流動的なタイポグラフィに`clamp()`を使用
- 最小の読みやすいサイズを設定（モバイルで14px）
- テキストスケーリングをテスト（200%にズーム）
- コンテナがテキストに合わせて拡張することを確認

### 国際化（i18n）

**テキストの拡張**：
- 翻訳のために30-40%のスペース予算を追加
- コンテンツに適応するflexbox/gridを使用
- 最も長い言語でテスト（通常ドイツ語）
- テキストコンテナに固定幅を避ける

```jsx
// ❌ 悪い例: 短い英語テキストを前提
<button className="w-24">Submit</button>

// ✅ 良い例: コンテンツに適応
<button className="px-4 py-2">Submit</button>
```

**RTL（右から左）サポート**：
```css
/* 論理プロパティを使用 */
margin-inline-start: 1rem; /* margin-leftではなく */
padding-inline: 1rem; /* padding-left/rightではなく */
border-inline-end: 1px solid; /* border-rightではなく */

/* またはdir属性を使用 */
[dir="rtl"] .arrow { transform: scaleX(-1); }
```

**文字セットサポート**：
- どこでもUTF-8エンコーディングを使用
- 中国語/日本語/韓国語（CJK）文字でテスト
- 絵文字でテスト（2-4バイトの場合がある）
- 異なるスクリプトの処理（ラテン、キリル、アラビアなど）

**日付/時刻フォーマット**：
```javascript
// ✅ 適切なフォーマットにIntl APIを使用
new Intl.DateTimeFormat('en-US').format(date); // 1/15/2024
new Intl.DateTimeFormat('de-DE').format(date); // 15.1.2024

new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD'
}).format(1234.56); // $1,234.56
```

**複数形**：
```javascript
// ❌ 悪い例: 英語の複数形を前提
`${count} item${count !== 1 ? 's' : ''}`

// ✅ 良い例: 適切なi18nライブラリを使用
t('items', { count }) // 複雑な複数形ルールを処理
```

### エラーハンドリング

**ネットワークエラー**：
- 明確なエラーメッセージを表示
- リトライボタンを提供
- 何が起きたか説明する
- オフラインモードを提供する（該当する場合）
- タイムアウトシナリオを処理する

```jsx
// リカバリー付きエラー状態
{error && (
  <ErrorMessage>
    <p>データの読み込みに失敗しました。{error.message}</p>
    <button onClick={retry}>再試行</button>
  </ErrorMessage>
)}
```

**フォームバリデーションエラー**：
- フィールド近くのインラインエラー
- 明確で具体的なメッセージ
- 修正の提案
- 不必要に送信をブロックしない
- エラー時にユーザー入力を保持する

**APIエラー**：
- 各ステータスコードを適切に処理する
  - 400: バリデーションエラーを表示
  - 401: ログインにリダイレクト
  - 403: 権限エラーを表示
  - 404: 未検出状態を表示
  - 429: レート制限メッセージを表示
  - 500: 汎用エラーを表示、サポートを提案

**グレースフルデグラデーション**：
- JavaScriptなしでコア機能が動作する
- 画像にaltテキストがある
- プログレッシブエンハンスメント
- サポートされていない機能のフォールバック

### エッジケースと境界条件

**空の状態**：
- リストにアイテムがない
- 検索結果がない
- 通知がない
- 表示するデータがない
- 明確な次のアクションを提供する

**ローディング状態**：
- 初回ロード
- ページネーションロード
- リフレッシュ
- 何がロードされているかを表示する（「プロジェクトを読み込んでいます...」）
- 長い操作の時間見積もり

**大量データ**：
- ページネーションまたは仮想スクロール
- 検索/フィルター機能
- パフォーマンスの最適化
- 10,000アイテムすべてを一度にロードしない

**同時操作**：
- 二重送信を防止する（ロード中にボタンを無効化）
- 競合状態を処理する
- ロールバック付きの楽観的更新
- コンフリクト解決

**権限状態**：
- 閲覧権限なし
- 編集権限なし
- 読み取り専用モード
- 理由の明確な説明

**ブラウザ互換性**：
- モダン機能のポリフィル
- サポートされていないCSSのフォールバック
- 機能検出（ブラウザ検出ではなく）
- ターゲットブラウザでテスト

### 入力のバリデーションとサニタイズ

**クライアントサイドバリデーション**：
- 必須フィールド
- フォーマットバリデーション（メール、電話、URL）
- 長さ制限
- パターンマッチング
- カスタムバリデーションルール

**サーバーサイドバリデーション**（常に）：
- クライアントサイドのみを信頼しない
- すべての入力をバリデートしサニタイズする
- インジェクション攻撃から保護する
- レート制限

**制約の処理**：
```html
<!-- 明確な制約を設定 -->
<input
  type="text"
  maxlength="100"
  pattern="[A-Za-z0-9]+"
  required
  aria-describedby="username-hint"
/>
<small id="username-hint">
  英数字のみ、最大100文字
</small>
```

### アクセシビリティの耐障害性

**キーボードナビゲーション**：
- すべての機能がキーボードでアクセス可能
- 論理的なタブ順序
- モーダルでのフォーカス管理
- 長いコンテンツのスキップリンク

**スクリーンリーダーサポート**：
- 適切なARIAラベル
- 動的変更のアナウンス（ライブリージョン）
- 説明的なaltテキスト
- セマンティックHTML

**モーション感度**：
```css
@media (prefers-reduced-motion: reduce) {
  * {
    animation-duration: 0.01ms !important;
    animation-iteration-count: 1 !important;
    transition-duration: 0.01ms !important;
  }
}
```

**ハイコントラストモード**：
- Windowsハイコントラストモードでテスト
- 色だけに頼らない
- 代替のビジュアルキューを提供する

### パフォーマンスの耐障害性

**低速接続**：
- プログレッシブ画像ローディング
- スケルトンスクリーン
- 楽観的UIの更新
- オフラインサポート（Service Worker）

**メモリリーク**：
- イベントリスナーのクリーンアップ
- サブスクリプションのキャンセル
- タイマー/インターバルのクリア
- アンマウント時のペンディングリクエストの中断

**スロットリングとデバウンス**：
```javascript
// 検索入力のデバウンス
const debouncedSearch = debounce(handleSearch, 300);

// スクロールハンドラーのスロットル
const throttledScroll = throttle(handleScroll, 100);
```

## テスト戦略

**手動テスト**：
- 極端なデータでテスト（非常に長い、非常に短い、空）
- 異なる言語でテスト
- オフラインでテスト
- 低速接続でテスト（3Gにスロットル）
- スクリーンリーダーでテスト
- キーボードのみのナビゲーションでテスト
- 古いブラウザでテスト

**自動テスト**：
- エッジケースのユニットテスト
- エラーシナリオの統合テスト
- クリティカルパスのE2Eテスト
- ビジュアルリグレッションテスト
- アクセシビリティテスト（axe、WAVE）

**重要**: 堅牢化は予想外のことを予期することです。実際のユーザーはあなたが想像もしないことをします。

**絶対にしてはいけないこと**：
- 完璧な入力を前提とする（すべてをバリデートする）
- 国際化を無視する（グローバル向けにデザインする）
- エラーメッセージを汎用的に残す（「エラーが発生しました」）
- オフラインシナリオを忘れる
- クライアントサイドバリデーションのみを信頼する
- テキストに固定幅を使用する
- 英語の長さのテキストを前提とする
- 1つのコンポーネントのエラーでインターフェース全体をブロックする

## 堅牢化の検証

エッジケースで徹底的にテストする：

- **長いテキスト**: 100文字以上の名前を試す
- **絵文字**: すべてのテキストフィールドで絵文字を使用
- **RTL**: アラビア語やヘブライ語でテスト
- **CJK**: 中国語/日本語/韓国語でテスト
- **ネットワークの問題**: インターネットを無効化、接続をスロットル
- **大量データ**: 1000以上のアイテムでテスト
- **同時アクション**: 送信を10回素早くクリック
- **エラー**: APIエラーを強制、すべてのエラー状態をテスト
- **空**: すべてのデータを削除、空の状態をテスト

覚えておいてください：あなたはデモの完璧さではなく、プロダクションの現実のために堅牢化しています。ユーザーが変なデータを入力し、フロー途中で接続を失い、予想外の方法で製品を使用することを予期してください。すべてのコンポーネントに耐障害性を組み込んでください。
