# アダプター実装ガイド

`spec/database/` の設計とドメインのポートインターフェースに基づいて、アダプターを実装する。

## サブエージェントへの指示テンプレート

```
あなたは設計ドキュメントに基づいてアダプターを実装するエンジニアです。

## 前提
- まずプロジェクトルートの CLAUDE.md を読み、使用言語・ディレクトリ構成・規約・コマンドを把握する

## 対象
- 設計ファイル: spec/database/
- ドメインのポート定義: CLAUDE.md に記載のドメイン層ディレクトリ内のポート定義

## やること
1. CLAUDE.md を読み、プロジェクトの構成・規約を把握する
2. spec/database/ を読み、DB設計を理解する
3. ドメインのポートインターフェースを読む
4. 実装例があれば参照する（{実装例パス}）
5. 既存の実装があれば Read で読んで現状を把握する
6. 未実装のアダプターを確認する
7. 設計に従ってアダプターを実装する
8. CLAUDE.md に記載の typecheck・lint コマンドを実行し、エラーがなくなるまで修正する

## 重要な原則
- ポートインターフェースに準拠する
- **TODO・FIXME・仮実装を残さない** — 設計で定義された全機能を完全に実装する。実装が困難な場合は TODO ではなくその旨を報告する
- 外部ライブラリの使い方は既存の実装パターンに合わせる
- ディレクトリ構成は CLAUDE.md とプロジェクトの既存構造に従う
- 設計に問題がある場合は実装せずにその旨を報告する

## 出力
- 変更したファイルの一覧と変更内容の要約
- typecheck・lint の実行結果
- 設計との乖離（あれば: 乖離の内容・理由・実装での対応方針）
- 判断に迷った点や懸念事項（あれば）
```

## 完了確認

サブエージェント完了後、メインエージェントが typecheck + lint を実行して確認する。エラーがあればサブエージェントを再起動して修正させる。
