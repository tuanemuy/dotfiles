# spec/ 出力フォーマット

## ディレクトリ構成

```
spec/
├── overview.md                 # サービス概要
├── sitemap.md                  # 画面一覧・サイト構造
├── auth.md                     # 認証・認可の仕様（認証が必要な場合のみ）
├── components.md               # 共通UIコンポーネント
├── pages/
│   ├── {page-name}.md          # 各画面の詳細仕様
│   └── ...
└── flows/
    ├── {flow-name}.md          # ユーザーフローの詳細
    └── ...
```

## 各ファイルのテンプレート

### overview.md

```markdown
# {サービス名}

## 概要

{サービスの目的と概要を1-3文で}

## ターゲットユーザー

{想定されるユーザー層}

## 主要機能

- {機能1}
- {機能2}
- ...

## 調査情報

- 調査日: {YYYY-MM-DD}
- 対象URL: {url}
- 調査範囲: {全体 or 特定セクション}
- 未調査箇所: {あれば記載}
```

### sitemap.md

```markdown
# サイトマップ

## 画面一覧

| 画面名 | URL パターン | セクション | 認証 | 概要 |
|--------|-------------|-----------|------|------|
| トップ | / | - | 不要 | ランディングページ |
| ログイン | /login | 認証 | 不要 | ログインフォーム |
| ダッシュボード | /dashboard | メイン | 必要 | ユーザーのホーム画面 |

## サイト構造

{セクションごとのツリー構造}

```
トップ (/)
├── 認証
│   ├── ログイン (/login)
│   └── 新規登録 (/signup)
├── メイン（要ログイン）
│   ├── ダッシュボード (/dashboard)
│   └── 設定 (/settings)
└── 公開ページ
    ├── 料金 (/pricing)
    └── ヘルプ (/help)
```

## 画面遷移

{主要な遷移関係。すべてを網羅する必要はなく、重要な関係のみ}
```

### auth.md

```markdown
# 認証・認可

## 認証方式

{メール/パスワード、OAuth、SSO 等}

## 認証フロー

{ログインの手順}

## 認可レベル

| レベル | アクセス可能な画面 | 備考 |
|-------|------------------|------|
| 未認証 | トップ、ログイン、料金 | - |
| 一般ユーザー | ダッシュボード、設定 | - |

## セッション管理

{判明した範囲で: タイムアウト、リフレッシュ、ログアウト動作}
```

### pages/{page-name}.md

```markdown
# {画面名}

- URL: {url}
- 認証: {必要 / 不要}
- セクション: {所属セクション}

## 目的

{この画面が何のためにあるか}

## レイアウト

{画面の大まかな構成}

## 入力項目

| 項目名 | 型 | 必須 | バリデーション | 初期値 | 備考 |
|--------|-----|------|--------------|--------|------|
| メールアドレス | email | Yes | 形式チェック | - | placeholder: "you@example.com" |
| パスワード | password | Yes | 8文字以上 | - | 強度インジケーターあり |

## アクション

| ラベル | 種類 | 動作 | 備考 |
|--------|------|------|------|
| ログイン | ボタン（primary） | フォーム送信 → ダッシュボードへ遷移 | disabled: 入力不完全時 |
| パスワードを忘れた | リンク | /forgot-password へ遷移 | - |

## 状態・表示切替

{タブ、モーダル、条件付き表示などがあれば}

| 条件 | 表示変化 |
|------|---------|
| ログイン失敗 | エラーメッセージ「メールアドレスまたはパスワードが正しくありません」 |

## スクリーンショット

{該当するスクリーンショットの相対パス}
```

### flows/{flow-name}.md

```markdown
# {フロー名}

## 目的

{このフローが達成すること}

## 前提条件

- {ログイン済み、特定の権限、等}

## ステップ

| # | 画面 | 操作 | 結果 |
|---|------|------|------|
| 1 | トップ | 「新規登録」をクリック | 新規登録画面へ遷移 |
| 2 | 新規登録 | メールアドレスを入力 | - |
| 3 | 新規登録 | パスワードを入力 | 強度インジケーター表示 |
| 4 | 新規登録 | 「登録」をクリック | 確認メール送信画面へ遷移 |

## 分岐

| 分岐点 | 条件 | 結果A | 結果B |
|--------|------|-------|-------|
| ステップ4 | メール未登録 | 確認メール画面 | 「既に登録済み」エラー |

## エラーケース

| 操作 | エラーメッセージ | 復帰方法 |
|------|----------------|---------|
| パスワード3文字で送信 | 「8文字以上で入力してください」 | パスワード再入力 |

## スクリーンショット

{各ステップのスクリーンショットの相対パス}
```

### components.md

```markdown
# 共通UIコンポーネント

## ナビゲーション

{ヘッダー、サイドバー、フッターの構成}

### グローバルヘッダー
- 表示要素: {ロゴ, ナビリンク, ユーザーメニュー}
- 認証状態による表示切替: {未ログイン時: ログイン/登録ボタン, ログイン時: ユーザーアイコン}

## 共通パーツ

{複数画面で使われるUI部品}

| コンポーネント | 使用箇所 | 概要 |
|--------------|---------|------|
| 検索バー | ヘッダー、一覧画面 | インクリメンタルサーチ対応 |
| ページネーション | 一覧画面全般 | 前後+ページ番号 |
| トースト通知 | 全画面 | 操作成功/失敗のフィードバック |

## モーダル・ダイアログ

{確認ダイアログ、詳細表示モーダル等}

| モーダル名 | トリガー | 内容 |
|-----------|---------|------|
| 削除確認 | 削除ボタン押下 | 「本当に削除しますか？」+ 確認/キャンセル |
```
